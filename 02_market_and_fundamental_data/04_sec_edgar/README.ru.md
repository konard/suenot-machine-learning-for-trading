# Как работать с фундаментальными данными

Комиссия по ценным бумагам и биржам США (SEC) требует от американских эмитентов, то есть зарегистрированных компаний и ценных бумаг, включая взаимные фонды, подавать три квартальных финансовых отчета (форма 10-Q) и один годовой отчет (форма 10-K), в дополнение к различным другим требованиям по регулятивной отчетности.

С начала 1990-х годов SEC сделала эти отчеты доступными через свою систему Electronic Data Gathering, Analysis, and Retrieval (EDGAR). Они составляют основной источник данных для фундаментального анализа акций и других ценных бумаг, таких как корпоративный кредит, где стоимость зависит от бизнес-перспектив и финансового здоровья эмитента.

## Автоматизированная обработка с использованием разметки XBRL

Автоматизированный анализ регулятивных отчетов стал намного проще после того, как SEC представила XBRL, бесплатный, открытый и глобальный стандарт для электронного представления и обмена бизнес-отчетами. XBRL основан на XML; он опирается на [таксономии](https://www.sec.gov/dera/data/edgar-log-file-data-set.html), которые определяют значение элементов отчета и сопоставляются с тегами, которые выделяют соответствующую информацию в электронной версии отчета. Одна такая таксономия представляет Общепринятые принципы бухгалтерского учета США (US GAAP).

SEC ввела добровольную подачу XBRL в 2005 году в ответ на бухгалтерские скандалы, прежде чем потребовать этот формат для всех подателей с 2009 года, и продолжает расширять обязательное покрытие на другие регулятивные отчеты. SEC ведет веб-сайт, который перечисляет текущие таксономии, формирующие содержание различных отчетов и которые могут использоваться для извлечения конкретных элементов.

Существует несколько способов отслеживания и доступа к фундаментальным данным, сообщаемым в SEC:
- В рамках [EDGAR Public Dissemination Service](https://www.sec.gov/oit/announcement/public-dissemination-service-system-contact.html) (PDS) электронные потоки принятых отчетов доступны за плату.
- SEC обновляет [RSS-каналы](https://www.sec.gov/structureddata/rss-feeds-submitted-filings) каждые 10 минут, которые перечисляют структурированные раскрытия.
- Существуют публичные [индексные файлы](https://www.sec.gov/edgar/searchedgar/accessing-edgar-data.htm) для получения всех отчетов через FTP для автоматизированной обработки.
- Наборы данных финансовой отчетности (и примечаний) содержат разобранные XBRL данные из всех финансовых отчетов и сопутствующих примечаний.

SEC также публикует лог-файлы, содержащие [интернет-поисковый трафик](https://www.sec.gov/dera/data/edgar-log-file-data-set.html) для отчетов EDGAR через SEC.gov, хотя и с шестимесячной задержкой.

## Построение временного ряда фундаментальных данных

Объем данных в наборах данных [Financial Statement and Notes](https://www.sec.gov/dera/data/financial-statement-and-notes-data-set.html) состоит из числовых данных, извлеченных из основных финансовых отчетов (баланс, отчет о прибылях и убытках, денежные потоки, изменения в капитале и совокупный доход) и сносок к этим отчетам. Данные доступны начиная с 2009 года.

Папка [04_sec_edgar](.) содержит ноутбук [edgar_xbrl](edgar_xbrl.ipynb) для загрузки и парсинга данных EDGAR в формате XBRL, а также создания фундаментальных метрик, таких как коэффициент P/E, путем объединения данных финансовой отчетности и данных о ценах.
